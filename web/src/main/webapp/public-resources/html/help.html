Baseline Atlas of Gene Expression: Help

About Baseline Atlas

Baseline Atlas aims primarily at representing information about which gene product is present (and at what abundance) in which "normal" condition (e.g., tissue, cell type). In the first instance Baseline Atlas will consist of individual well understood and curated RNA–seq experiments from ArrayExpress, such as:

  • Illumina Body Map
  • Transcriptome of DBAxC57BL/6J mice
  • RNA–seq of long coding and long non coding RNAs from ENCODE cell lines
  • RNA–seq of 6 tissues from 10 species to investigate the evolution of gene expression levels in mammalian organs

The above experiments have been re–analyzed using an in–house RNA–seq processing pipeline, ensuring the highest quality of the original data and of subsequent experiment metadata curation. The resulting data matrix stores gene expression profiles across the analyzed experimental factor values as a set of normalized expression levels (FPKMs). You can search the data by a combination of genes conditions, experimental factor values, and a minimum FPKM cutoff. See below for more details on searching.

Filtering Baseline Atlas Data

Some experiments have more than one experimental variable (factor). For example, in one experiment, three different types of RNA were extracted from six cellular components from 23 human cell lines. Results for these experiments are limited to a certain combination of variables, e.g. showing expression of total RNA in the whole cell. To limit the search results differently, use the "Change filters" menu. In a 3–factor experiment, you can choose one value from any two of the factors, and then search among the values for the remaining factor. E.g., if you choose to limit to poly(A) RNA extracted from the cytosol, you can then search among the cell lines for which this data is available.

Searching Baseline Atlas

Searching by genes

You can search by Ensembl gene symbols (e.g. SAA4), identifiers (e.g. ENSG00000148965) or biotypes (e.g. protein_coding), UniProt accessions (e.g. P35542), GO ("phosphoglycerate kinase activity") or InterPro terms (e.g. phosphatase). A space–separated list of gene attributes will bring back genes that match at least one of the attributes in the query.

Searching by experimental factor values

You can search for any combination of experimental factor (e.g. organism part or cell line) values available:

  • within a single–factor experiment, e.g. Illumina Body Map, or
  • within a pre–selected subset of a multi–factorial experiment, e.g. RNA–seq of long coding and long non coding RNAs from ENCODE cell lines

When you click on the experiment factor value input box, a dropdown menu appears, containing the list of all factor values available for selection:

Select one or more factor values to search with, or start typing to see suggestions.

Specific vs. non–specific search

The default Baseline Atlas search reports first genes with more "specific" expression (with the Specific option selected by default). What we mean by this is outlined below:

  • If the query contains no experimental factor values, the Specific search will report genes expressed in just one experimental factor value, then genes expressed in just two factor values, then three and so on, reporting any genes expressed in all the available factor values at the end of the list of results. Within each group of genes expressed in N factor values only, genes are sorted by the highest average FPKM across the N factor values in which they are expressed.
  • If the query contains at least one experimental factor value, the Specific search will promote genes with higher FPKM across the queried factor values, and at the same time penalize genes with the higher FPKM across the non–queried factor values. In other words, genes with the greatest difference in expression between queried and non–queried tissues will be pushed towards the top of the list.

If the Specific option is not selected, Baseline Atlas search will ignore specificity of expression when ordering the results. In particular:

  • If the query contains no experimental factor values, genes with the highest expression across all factor values will be reported first.
  • If the query contains at least one experimental factor value, genes with higher FPKM across the selected factor values are reported first (expression levels across the non–selected factor values is ignored in the ordering of results).

Specifying the minimum expression level

You can specify a minimum expression level to act as a cutoff, and only genes expressed above this level are displayed. This is to aid distinction between "background noise" and "real" gene expression. Our default FPKM cutoff is 0.5. To change it, enter a new value into the "Expression level cutoff" input box, or drag the slider until the "Expression level cutoff" input box shows your desired cutoff value. A histogram showing the number of genes expressed above a range of cutoffs is also available. This lets you see how many genes you are excluding by selecting a given cutoff. If you see that you're excluding a lot of genes, you may wish to decrease the cutoff. Clicking on the button toggles display of the histogram.

  • If you have not selected any experimental factors, the histogram shows the number of genes expressed in any factor value for each of the shown cutoffs.
  • If you have selected at least one factor value, the histogram shows the number of genes expressed in at least one of the selected factor values, for each of the shown cutoffs.

Heatmap Results Display

Results for your query (or the default query) are shown in a heatmap table. The heatmap rows and columns are labelled with gene symbols and experimental factor values, respectively. By default, the heatmap shows expressions by colour intensity, according to the gradient legend,, displayed above the heatmap. Greater colour intensity means higher relative gene expression. To see the exact FPKM for a given gene in a given factor value, click on the corresponding box. To reveal this for all genes, click on the button. This also reveals minimum and maximum FPKMs on each side of the colour gradient. The gradient shows intensities corresponding to expression levels for the top 50 (default) genes currently displayed (rather than all FPKM values for all genes returned by the query).

When displayed factor values are organism parts, a mouseover action over each heatmap column highlights thus selected tissue in the anatomogram on the left (if available). If the selected organism part is gender–specifc, and anatomogram is available for each gender (i.e. if you can see the button), you may need to switch to the gender to which a given tissue applies before seeing it highlighted.

Anatomogram

The anatomogram provides an attractive focal point for the tissues studied in a given experiment. As a reciprocal functionality for column mouseover in the heatmap, the mouseover action over tissues visualised in the anatomogram highlights the corresponding column in the heatmap table.

Downloading Query Results

Only the top 50 genes resulting from your search are displayed on the page by default. Click on the button to download the full results of your query in tab–delimited format with no ordering.

Experiment Design Page

The Experiment Design page (e.g. Illumina Body Map) shows RNA–seq processing run acessions (from ENA), along with their coresponding biological sample characteristcs and factor values. You can access it from the experiment page by clicking the button. By default, only the runs used in Baseline Atlas are shown, although the dataset these were taken from may contain more. To see all runs in this dataset, including those not used for Baseline Atlas, uncheck Show Analysed only. You can sort the experiment design table by clicking on a column. To then sort by another column while retaining the ordering of the first column you clicked, hold down the shift key and click another column. The first selected column forms the primary sort order and the shift+click–ed columns form secondary, tertiary, etc. sort orders in turn.

Please note that a column may be in three states:

  1. sorted in ascending order
  2. sorted in descending order
  3. unsorted

In single–column sort mode (without holding shift), clicking on a column heading jumps between ascending and descending. In multi–column sort mode (shift+click), clicking on a column heading alternates between ascending, descending and unsorted. If you shift+click a column to sort by it and then wish to undo this, just shift+click it again until the column is unsorted ().

The experiment design table is also searchable — entering a keyword automatically selects the subsection of the table that matches the keyword. Click on the button on the right–hand side to download the full results of the current query (with no ordering applied) in tab–delimited format.

Analysis Methods Page

The Analysis Methods page (e.g. Illumina Body Map) lists the data analysis methods applied to the raw experimental data in FASTQ format, in the order in which the methods were applied, to arrive at the gene expression levels shown in Baseline Atlas. You can access it from the experiment page by clicking the button.

We would also be grateful for any feedback on the analysis methods we have adopted — we are passionate about not only about intuitive data presentation, the highest level of experimental metadata curation, but also about the quality and biological validity of the expression data presented in Baseline Atlas.
