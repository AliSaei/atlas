Baseline Atlas of Gene Expression - Help

About Baseline Atlas

Baseline Atlas aims primarily at representing information about which gene product is present (and at what abundance) in which normal condition (e.g., tissue, cell type). In the first instance Baseline Atlas will consist of individual well understood and curated RNA-Seq experiments, such as:

  • Illumina Body Map
  • Transcriptome of DBAxC57BL/6J mice
  • RNA-seq of long coding and long non coding RNAs from ENCODE cell lines
  • RNA-seq of 6 tissues from 10 species to investigate the evolution of gene expression levels in mammalian organs

The above experiments have been re-analyzed using an in-house RNA-seq processing pipeline, ensuring the highest quality of the original data and of subsequent experiment metadata curation. The analysis methods used are shown below (TODO). The resulting data matrix stores gene expression profiles across the analyzed experimental factor values as a set of expression counts (FPKMs). The user is able to search the data  by a combination of genes conditions, experimental factor values, and an FPKM cutoff. All expressions above the selected cutoff are considered to be 'real', and the expression on or below the FPKM cutoff are treated as 'background'.A default cutoff is pre-selected. A histogram of the number of genes expressed above a given cutoff for a range of FPKM cutoffs is also available - to help the user choose the most appropriate cutoff for their query of interest.

Searching Baseline Atlas

  • Gene Conditions
    • You can search by Ensembl gene symbols (e.g. SAA4) , identifiers (e.g. ENSG00000148965) or biotypes (e.g. protein_coding), UniProt accessions (e.g. P35542), GO ("phosphoglycerate kinase activity") or InterPro terms (e.g. phosphatase). A space-separated list of gene conditions will bring back genes that match at least one of the conditions in the query.
  • Experimental Factor Values
    • You can search for any combination of experimental factor (e.g. organism part or cell line) values available
      • within a single-factor experiment, e.g. Illumina Body Map, or
      • within a pre-selected subset of a multi-factorial experiment, e.g. RNA-seq of long coding and long non coding RNAs from ENCODE cell lines
    • When you click on the experiment factor value input box, a dropdown appears, containing the list of all factor values available for selection:
    • Specific vs Non-specific search.
      • The default Baseline Atlas search reports first genes more specifically expressed (with the Specific option selected by default).
        • If the query contains no experimental factor values, the Specific search will report genes expressed in just one experimental factor value, then genes expressed in just two factor values, then three and so on, reporting any genes expressed in all the available factor values at the end of the list of results. Within each group of genes expressed in N factor values only, genes are sorted by the highest average FPKM across the N factor values in which they are expressed.
        • If the query contains at least one experimental factor value, the Specific search will reward genes with higher average FPKM across the queried factor values, and at the same time penalize genes with the higher FPKM across the non-queried factor values. In other words,  genes with the highest difference in average expression between queried and non-queried tissues will be shown first.
      • If the Specific option is not selected, Baseline Atlas search will ignore specificity of expression when ordering the results. In particular:
        • If the query contains no experimental factor values, genes with the highest average expression across all the available tissues will be reported first.
        • If the query contains at least one experimental factor value, genes with the higher average FPKM across the selected factor values are reported first (and expression levels across the non-selected factor values is ignored in the ordering of results)
  • Specifying the Expression level cutoff
    • The expression level cutoff is intended to discriminate between the 'real' (i.e. above the cutoff) and the 'background' (i.e on or below the cutoff) expression for each gene. The default FPKM cutoff is 0.5, but the user is able to selected a different cutoff if need be. This can be done by inputting a different value into the 'Expression level cutoff' input box, or moving the slider along until the 'Expression level cutoff' input box shows the intended cutoff value. A histogram showing the number of genes expressed above selected cutoffs is also available - to help the user decided how many genes they are excluding by selecting a given cutoff. Clicking on the button displays/hides a histogram.
      • If the user has not selected any experimental factors, the histogram shows the number of genes expressed in any factor value for each of the shown cutoffs.
      • If the user has selected at least one factor value, the histogram shows the number of genes expressed in at least one of the selected factor values, for each of the shown cutoffs.

Heatmap Results Display

  • The heatmap rows and columns are labelled by gene symbols and experimental factor values respectively. By default, the heatmap shows represents expressions by colour intensity, according to the gradient legend,, displayed above the heatmap. Clicking on the button, displays all the FPKM values in the heatmap as well as the minimum and the maximum FPKM on each side of the colour gradient. From this you can see that gradient encodes colour intensities corresponding to the FPKM values for the top 50 (default) genes currently displayed to the user (rather than all FPKM values for all genes returned by the query). In Addition, the user is able to click on each heatmap cell to reveal the underlying FPKM value. Clicking on the cell with the FPKM displayed in turn hides the FPKM again.
  • When displayed factor values are organism parts, a mouseover action over each heatmap column highlights thus selected tissue in the anatomogram on the left (if available). If the selected organism part is gender-specifc, and anatomogram is available for each gender (i.e. if you can see the button), you may need to switch to the gender to which a given tissue applies before seeing it highlighted.

Anatomogram

  • The anatomogram provides an attractive focal point for the tissues studied in a given experiment. As a reciprocal functionality for column mouseover in the heatmap, the mouseover action over tissues visualised in the anatomogram highlights the corresponding column in the heatmap table.

Downloading Query Results

  • Clicking on the button, downloads the full results of the current query in tab-delimited format with no ordering.

Experiment Design Page

The Experiment Design page (e.g. Illumina Body Map) shows RNA-seq processing run acessions, along with their coresponding sample characteristcs and factor values. By default, only the runs that have been re-analyzed for Baseline Atlas are shown. To see all runs, the Show Analysed only checkbox needs to be unselected. The experiment design table is sortable by clicking on individual columns. Sorting by multiple columns is also possible, by holding down the shift key and selecting more than column. In this case, the first selected column forms the primary sort order and the subsequently selected columns form secondary, tertiary, etc. sort orders in turn.

Please note that a column being sorted may be in three states:

  1. sorted in ascending order
  2. sorted in descending order and
  3. usorted

In the single-column sort mode (the shift key not pressed), clicking on a column heading jumps between 1. and 2.
In the multiple-column sort mode (the shift key pressed), clicking on a column heading alternates between 1., 2. and 3.
Essentially, in the multiple-column sort mode, leaving one of the columns in state 3. removes it from the current sort operation.

The experiment table is also searchable - entering a keyword automatically selects the subsection of the experiment design table that matches the keyword.

Finally, clicking on the button on the right-hand side, downloads the full results of the current query (with no ordering applied) in tab-delimited format.

Analysis Methods Page

The Analysis Methods page (e.g. Illumina Body Map) lists data analysis methods applied to the original experiment data in FASTQ, in the order in which the methods were applied, to arrive at the gene expression levels shown on in Baseline Atlas.

Providing Feedback

Baseline Atlas is still a prototype and thus more then ever we rely and value user feedback on what we are doing right, what does not work or what could be achieved more intuitively to the user. We would also be grateful for any feedback on the analysis methods we adopted - we are passionate about not only about intuitive data presentation, the highest level of experimental metadata curation, but also about the quality and biological validity of the expression data presented in baseline Atlas.
